## 一、拖拽功能重构

### 1.1 问题背景
- 原先使用 dnd-kit 库处理拖拽，但与原生 HTML5 拖拽事件冲突
- 右侧面板标签无法拖拽到中间集合区
- 多次尝试修复均导致页面闪退或功能失效

### 1.2 最终解决方案
完全移除 dnd-kit，使用纯原生 HTML5 拖拽 API

#### 技术实现
1. **RightPanel 拖拽开始**
   - 使用 onDragStart 事件
   - 通过 dataTransfer.setData() 设置 JSON 数据
   - 设置 effectAllowed = 'move'
   - 添加全局标记 __TOBY_EXTERNAL_DRAG__ 供 MainContent 检测

2. **MainContent 拖拽接收**
   - 使用文档级别 document.addEventListener 监听 dragover 和 drop 事件
   - 使用 closest('[data-collection-id]') 检测用户实际放置的目标集合
   - 降级处理：未放置到具体集合时默认添加到第一个

3. **数据处理**
   - 解析 JSON 数据获取标签信息
   - 创建新 Card 对象
   - 更新 allCollections 数组
   - 调用 onCollectionsChange 触发渲染和存储

## 二、Space 数据持久化修复

### 2.1 问题描述
- 切换 Space 时，之前 Space 的集合数据丢失
- 除非用户手动删除，否则数据不应该丢失

### 2.2 原因分析
- handleAddCollection、handleDeleteCollection、handleUpdateCollection 使用了 collections（当前 Space 的过滤数据）
- 应该使用 allCollections（所有 Space 的完整数据）

### 2.3 修复方案
将所有集合操作函数中的 collections 替换为 allCollections

---

## 三、Popup.js 拖拽防崩溃重构

### 3.1 需求来源
用户要求 popup.js 实现稳定的拖拽功能

### 3.2 实现方案

1. **渲染节流**
   - 定义 render() 函数使用 requestAnimationFrame
   - 内部维护 rafId 和 isRenderScheduled 状态

2. **拖拽防抖**
   - 使用 isDragging 变量作为开关
   - render() 函数开头检查 isDragging，拖拽中禁止刷新

3. **事件隔离**
   - handleDrop 第一行必须是 e.preventDefault() 和 e.stopPropagation()
   - 数据处理逻辑完全封装在 try-catch 中
