## 实现过程中的问题

### 1. CardItem 组件参数缺失
- **问题**: 添加拖拽和右键菜单功能后，需要向 CardItem 传递新的参数
- **解决**: 添加了 `collectionId`、`onEdit`、`onMove` 参数

### 2. SortableCollection 组件参数缺失
- **问题**: CardItem 新增的回调函数需要从 SortableCollection 传递到 MainContent
- **解决**: 在 SortableCollection 组件中添加了 `onEditCard` 和 `onMoveCard` 参数

### 3. DndContext 处理逻辑
- **问题**: 需要区分拖拽的是标签页、卡片还是集合排序
- **解决**: 通过 `active.data.current?.type` 判断数据类型，分别处理

### 4. X 按钮无响应问题（2026-03-19）
- **问题**: 点击卡片上的 X 删除按钮无响应
- **原因**: CardItem 组件使用 `useDraggable`，`...listeners` 绑定到整个元素，捕获了 pointer 事件导致 click 事件无法触发
- **解决**: 将 `...attributes` 和 `...listeners` 移到卡片内部的可拖拽区域（标题和图标部分），使 X 按钮不受拖拽事件影响
- **修改位置**: `src/components/MainContent.tsx` CardItem 组件

### 5. 卡片点击无法打开链接（2026-03-20）
- **问题**: 点击卡片无法打开链接
- **原因**: useDraggable 默认会捕获 pointer 事件，点击时也触发拖拽，导致 handleClick 没有正确执行
- **解决**: 添加 `activationConstraint: { distance: 5 }` 让拖拽需要移动 5px 才能激活，这样普通的点击就不会触发拖拽
- **修改位置**: `src/components/MainContent.tsx` CardItem 组件 useDraggable 配置

## 待优化项

### 1. 右键菜单定位
- 当前使用 `fixed` 定位，菜单可能出现在视口外
- 建议：添加边界检测，确保菜单在视口内显示

### 2. 拖拽视觉反馈
- 拖拽 Card 时没有专门的 DragOverlay
- 建议：添加卡片拖拽时的视觉预览

### 3. 移动卡片功能
- 当前移动卡片通过右键菜单实现
- 未来可以考虑：支持跨 Space 移动（当前只支持同 Space 内移动）

### 4. 关闭标签页的替代方案
- 当前完全移除了关闭标签页的功能
- 用户可能需要选择性的关闭
- 建议：可通过快捷键（如 Alt+拖拽）来切换是否关闭原标签页

### 5. AI 功能优化
- 当前只支持分析已打开的标签页
- 建议：支持未打开的 URL 直接分析（需要先打开标签页或使用服务端代理）
- 当前批量分析功能已实现但未在 UI 中暴露入口

### 6. 动态导入警告
- 构建时出现动态导入警告
- 原因：ai.ts 同时被静态和动态导入
- 建议：统一导入方式

## 测试建议

1. **Card 拖拽测试**
   - 创建多个 Collection
   - 将 Card 从 Collection A 拖拽到 Collection B
   - 验证 Card 正确移动且原位置删除

2. **右键菜单测试**
   - 右键点击 Card
   - 测试编辑功能（修改名称和 URL）
   - 测试移动功能（移动到其他 Collection）
   - 测试删除功能

3. **拖拽标签页测试**
   - 从右侧面板拖拽标签页到 Collection
   - 验证原标签页仍然打开
   - 验证 Collection 中新增了 Card

4. **AI 分析测试**
   - 配置 MiniMax API Key
   - 点击卡片的 AI 分析按钮
   - 验证分析结果展示
   - 验证同步到描述功能

5. **一键全打开测试**
   - 点击集合标题栏的「打开」按钮
   - 验证所有链接在标签页中打开
