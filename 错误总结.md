# Toby Chrome Extension 错误总结

## 一、拖拽功能问题

### 1.1 dnd-kit 与原生拖拽冲突

**问题描述**
- 使用 dnd-kit 库处理内部拖拽（集合排序、卡片移动）
- 但 dnd-kit 拦截了所有拖拽事件，导致原生 HTML5 拖拽无法触发

**尝试的解决方案**

1. 移除 useDroppable - 失败
2. 添加 PointerSensor 的 activationConstraint - 失败
3. 设置 autoScroll={false} - 失败
4. 使用原生 React onDragOver/onDrop 事件 - 部分成功，但第二次拖拽失败

**最终解决方案**
完全移除 dnd-kit，使用纯原生 HTML5 拖拽 API

---

### 1.2 拖拽事件无法到达目标区域

**问题描述**
- 从右侧面板拖拽标签到中间集合区时，drop 事件不触发

**原因分析**
- DndContext 和 SortableContext 拦截了原生拖拽事件
- React 合成事件与原生事件冲突

**解决方案**
使用文档级别的 document.addEventListener 监听拖拽事件：
```javascript
document.addEventListener('dragover', handleDocumentDragOver);
document.addEventListener('drop', handleDocumentDrop);
```

---

### 1.3 总是添加到第一个集合

**问题描述**
- 有多个集合时，无论拖到哪个集合，都只添加到第一个集合

**原因分析**
- 没有正确检测用户实际放置的目标集合

**解决方案**
使用 closest() 方法查找目标元素：
```javascript
const targetElement = (e.target as HTMLElement).closest('[data-collection-id]');
const collectionId = targetElement?.getAttribute('data-collection-id');
```

---

## 二、数据持久化问题

### 2.1 Space 切换数据丢失

**问题描述**
- 切换 Space 时，之前添加的集合数据丢失

**原因分析**
- handleAddCollection、handleDeleteCollection、handleUpdateCollection 使用了 collections（当前 Space 过滤后的数据）
- 应该使用 allCollections（所有 Space 的完整数据）

**解决方案**
将所有集合操作函数中的 collections 替换为 allCollections

---

## 三、TypeScript 编译错误

### 3.1 未使用的变量

**错误信息**
```
error TS6133: 'xxx' is declared but its value is never read.
```

**解决方案**
- 使用下划线前缀标记未使用的参数：`_e`
- 或者完全移除未使用的变量

---

### 3.2 重复代码块

**错误信息**
```
error TS1005: '}' expected.
```

**原因**
- 编辑代码时意外复制了重复的代码块
- 花括号不匹配

**解决方案**
- 使用 git diff 检查变更
- 仔细检查代码结构

---

## 四、Popup.js 问题

### 4.1 拖拽导致页面崩溃

**问题描述**
- popup.js 中的拖拽功能可能导致页面崩溃

**解决方案**
按照用户要求的架构重构：
1. 渲染节流 - 使用 requestAnimationFrame
2. 拖拽防抖 - 使用 isDragging 开关
3. 事件隔离 - try-catch 包裹
4. 禁止直接操作 DOM - 通过 state 更新驱动渲染

---

## 五、调试技巧

### 5.1 添加调试日志

在关键位置添加 console.log：
```javascript
console.log('[DragStart] 右侧标签开始拖拽:', { tabId, title, url });
console.log('[Document] Drop event captured');
console.log('[Document] Drop data:', dataStr);
```

### 5.2 检查事件触发

使用 Chrome 开发者工具：
1. 打开扩展面板
2. 打开 Console
3. 执行拖拽操作
4. 查看日志输出

### 5.3 检查 CSS 问题

使用 Chrome 开发者工具：
1. 检查元素是否有 pointer-events: none
2. 检查是否有 user-select: none
3. 检查元素是否有 z-index 遮挡

## 七、经验总结

### 7.1 避免 dnd-kit 与原生拖拽混用
dnd-kit 会拦截所有拖拽事件，如果需要支持外部拖拽，建议完全使用原生 API

### 7.2 使用 allCollections 进行数据操作
任何集合的增删改都应该基于完整数据（allCollections），而不是过滤后的数据（collections）

### 7.3 文档级事件监听
当 React 组件的事件监听不生效时，可以尝试使用 document 级别的事件监听

### 7.4 调试时先加日志
在排查问题时，先添加详细的 console.log，确认事件是否触发
